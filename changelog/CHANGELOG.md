# CHANGELOG

## Immigracious Project Change Log

This file serves as the index for all project changes. Each session's changes are logged in dated files within this folder.

---

## Log Format

Each changelog entry follows this structure:
- **File naming:** `CHANGELOG_YYYY-MM-DD_HH-MM.md` (24hr format, local time)
- **Content:** Tasks completed, decisions made, changes to PLAN.md

## How to Use This System

### When Starting a Session:
1. Check latest changelog for context
2. Review PLAN.md for current phase and tasks
3. Create new changelog file for this session

### When Ending a Session:
1. Log all completed tasks
2. Log all decisions made
3. Log any amendments to PLAN.md
4. Update this index

### When Amending PLAN.md:
1. Make the change in PLAN.md
2. Document the change in current session's changelog
3. Include: what changed, why, impact on timeline
